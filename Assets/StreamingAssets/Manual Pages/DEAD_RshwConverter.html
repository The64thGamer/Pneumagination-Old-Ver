DEAD_RshwConverter.cs

[WARNING: .rshw and equivalent files use the deprecated BinaryFormatter library, which is now deemed as a vulnerable format for attacks. Make sure you are loading these from a trusted source.]

DEAD_RshwConverter.cs is a DEAD Engine example script for converting showtapes from the RR Engine to the DEAD Engine.

This script is only usable through the Unity Editor.

Setting Up

The script requires a DEAD_Interface.cs be present and appended to the script in the editor.

Upon hitting play the "ConvertRshwAndInjectIntoInterface" bool can be pressed in the inspector. This will prompt the user to select a compatible RR Engine file. After selecting a file it will attempt to be processed and injected into the DEAD_Interface script.

Pressing the "saveFile" bool will save the new showtape as a file through a prompt.

Conversion Process

RR Engine file extensions are typically labeled .cshw, .sshw, .rshw, .nshw, .fshw, .tshw, and .mshw. None of the different extensions have any different file format and are simply to denote their stage type.

Be aware that RR Engine files are inherently unsafe, and to only attempt to convert them from trusted sources or ones you have made yourself from a non-modified RR Engine executable.

Be aware that some strange audio types may not convert properly when doing this, and may have to be replaced by a different file format. Programming signals should convert just fine however.

RR Engine files originally were titled and authored by their file name, signified by a dash (Showtape Name - Author.rshw). These parameters are saved into the internal Show Name and Author metadata of the new file format, allowing files to be named seperately.
